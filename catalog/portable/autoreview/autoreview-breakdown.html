How the autoreview skill works

`autoreview` is a closeout review workflow. It freezes a local, branch, or commit diff into a bounded review bundle, sends that bundle to one or more review engines, validates structured JSON, and exits clean only when no actionable findings remain.

1
Pick the targetDirty work, branch diff, or a single commit.
2
Build the bundleStatus, diffs, optional prompt notes, and datasets.
3
Run the engineCodex by default; Claude, Droid, Copilot, or a panel are optional.
4
Validate the reportSchema, path scope, required findings, and exit code.

Mental Model

The skill is mostly policy plus one Python helper. The helper owns repeatability; the skill text owns when and how an agent should use it.

Input

User or closeout trigger

A request for Codex review, Claude review, autoreview, second-model review, or a non-trivial code change that deserves a final check.

Scope

Target selection

The helper decides whether to review local dirty work, a branch against a base, or a single commit.

Context

Review bundle

Git status, stats, patches, optional notes, and optional datasets are packed into one prompt.

Review

Engine call

One engine, or an opt-in panel, produces a strict JSON report using the shared schema.

Closeout

Validated result

The helper filters, prints, writes optional outputs, waits for parallel tests, and exits 0 only for a clean result.

Files

The skill directory contains the human instructions, the actual automation, and a cross-platform smoke harness.

SKILL.md

Defines the trigger conditions, contract, target-picking guidance, panel syntax, helper paths, and final report expectations.

scripts/autoreview

The Python helper. It builds bundles, calls engines, parses JSON, validates findings, runs optional tests, and returns the final exit status.

scripts/test-review-harness*

Creates disposable repos with malicious or benign fixtures, then checks whether selected engines catch or avoid expected security findings.

Targeting

The target determines what git commands are used and which paths findings are allowed to reference.

Mode How it is chosen What gets reviewed Typical command
local Forced with --mode local, or automatic when the repo is dirty. Staged diff, unstaged diff, and untracked files not excluded by git. scripts/autoreview --mode local
branch Forced with --mode branch, or automatic on non-main clean branches. base...HEAD diff, where base comes from --base, PR metadata, or origin/main. scripts/autoreview --mode branch --base origin/main
commit Forced with --mode commit. git show for the selected commit ref. scripts/autoreview --mode commit --commit HEAD
Important: a clean local review means there was no local patch. For committed or pushed work, use branch or commit mode so the helper has a real diff to review.

Bundle Construction

Everything the reviewer sees is shaped into a single prompt before the engine starts.

Core bundle

  • local: git status, staged stat/patch, unstaged stat/patch, and untracked file contents.
  • branch: branch diff stat and patch against the selected base.
  • commit: commit metadata, stat, and patch from git show.
  • Large text fields are bounded so very large diffs are truncated rather than passed without limit.

Optional context

  • --prompt appends extra review instructions.
  • --prompt-file reads one or more instruction files.
  • --dataset embeds extra evidence files, such as logs or reproduction notes.
  • --no-web-search removes web search or web fetch permissions where supported.
You are a senior code reviewer.
Return exactly one JSON object.
Do not modify files.
Report only actionable defects introduced or exposed by this change.
Include security findings when concrete and actionable.

Engines

Codex is the default. Each engine is wrapped so the same schema and final validation path can be reused.

Codex

Runs codex exec with ephemeral mode, repo cwd, read-only sandbox, an output schema file, and an output-last-message file. Web search is enabled unless disabled.

Claude

Runs claude --print with JSON or stream JSON output and the shared schema. Default tools are read, grep, glob, web search, and web fetch.

Droid

Writes the prompt to a temporary file and calls droid exec with JSON output. It can disable tools with --disabled-tools *.

Copilot

Writes the prompt into a private temporary directory and asks Copilot to read it. It allows read-like tools and web fetch, with all URLs allowed when web search is on.

Command lookup

Bare commands are resolved from absolute PATH entries only. PATH entries inside the reviewed repo are skipped to avoid executing repo-provided binaries by accident.

Streaming

--stream-engine-output shows compact live activity while preserving buffered output for schema validation. Heartbeats are printed during long quiet periods.

Validation And Exit

The helper treats the model result as data, not as trusted prose.

Structured report schema

  • findings: array of title, body, priority, confidence, category, and code location.
  • overall_correctness: exactly patch is correct or patch is incorrect.
  • overall_explanation: concise explanation.
  • overall_confidence: number from 0 to 1.

Exit behavior

  • Exit 0 when there are no kept findings and the overall verdict is correct.
  • Exit nonzero when actionable findings remain or tests fail.
  • --expect-findings inverts success for harness tests.
  • --require-finding requires specific text in kept findings.
Scope filter: findings whose file path is outside the changed path set are printed as ignored and removed from the kept report. That keeps reviews focused, but it can hide cross-file issues if the reviewer points at an unchanged file.

Review Panels

Panels are opt-in and run multiple engines against one frozen bundle.

How reviewers are selected

--panel means Codex plus Claude unless --engine changes the first reviewer. --reviewers accepts comma-separated engines, and all expands to every supported engine.

scripts/autoreview --panel
scripts/autoreview --reviewers codex,claude
scripts/autoreview --reviewers codex:gpt-5.1:high,claude:sonnet:max

How reports are merged

Each reviewer validates independently. The panel merge deduplicates by file, line, category, and normalized title, then prefixes each finding body with the reviewer label.

--model codex=gpt-5.1
--thinking codex=high
--allow-partial-panel

Security And Usability Notes

These are the operational edges to understand before using the helper on sensitive or untrusted repositories.

Untracked files are included

Local mode reads untracked file contents into the bundle. That is useful for new files, but risky when a repo has accidental secrets or symlinks.

Branch mode fetches

Branch mode currently runs git fetch origin --quiet. That refreshes refs, but it also performs network activity and writes into .git.

Read-only is not privacy-only

Reviewer tools may inspect files and use web tools. Use --no-web-search and tool restrictions for private code or hostile diffs.

Parallel tests use a shell

--parallel-tests runs the supplied command through the platform shell by default. Treat that argument as trusted operator input.

Model output is advisory

The skill contract says every finding must be verified against the real code path before being accepted and fixed.

Stop on clean validation

When the helper exits 0 with no actionable findings, the skill tells the agent to stop instead of spending another review cycle for nicer wording.

Command Reference

The everyday commands are small; the helper carries the complicated parts.

Common review runs

# Dirty local work
catalog/portable/autoreview/scripts/autoreview --mode local

# Branch or PR work
catalog/portable/autoreview/scripts/autoreview --mode branch --base origin/main

# Single committed change
catalog/portable/autoreview/scripts/autoreview --mode commit --commit HEAD

Extra context and tests

# Add notes and evidence
catalog/portable/autoreview/scripts/autoreview \
  --mode branch \
  --base origin/main \
  --prompt-file /tmp/review-notes.md \
  --dataset /tmp/evidence.json

# Run tests alongside review
catalog/portable/autoreview/scripts/autoreview \
  --parallel-tests "cargo test -p skill-importer"
Final report shape: include the review command used, tests or proof run, accepted and rejected findings, and the final clean helper result or the reason any remaining finding was consciously rejected.
